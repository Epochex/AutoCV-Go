# AutoCV Go · 中文网申填充助手

AutoCV Go 是一款本地优先的 Chrome / Edge 浏览器扩展，面向校招、实习和社会招聘网申场景。

你只需维护一份简历资料。进入招聘页面后，AutoCV Go 会识别表单、匹配对应内容并生成填写预览，由你确认后写入网页。它不会自动投递，也不会替你点击提交按钮。

## 它能解决什么问题

不同公司的网申系统经常要求重复填写姓名、教育经历、实习经历、项目、论文和技能等信息。AutoCV Go 把这些内容集中保存，在不同招聘网站之间重复使用，减少复制、粘贴和反复修改的时间。

它适合：

- 经常填写中文校招、实习或社招申请的求职者；
- 希望统一维护多段教育、工作、项目和科研经历的用户；
- 希望使用自己的 DeepSeek、OpenAI 或兼容 API 辅助识别复杂表单的用户；
- 重视隐私，希望简历资料主要保存在本地浏览器中的用户。

## 核心功能

### 一份资料，多站复用

支持保存基本信息、教育、工作、实习、项目、科研与论文、开源贡献、技能、语言、奖项和自定义内容。也可以导入 PDF、Markdown 或 LaTeX 简历，审阅解析结果后再写入资料库。

### 扫描全部表单，确认后填写

插件会扫描当前网页中可访问的输入框、文本框、选择框、日期、文件上传和常见自绘控件。对于常见的自绘下拉框，插件会打开选项面板并选择与简历资料一致的值。扫描结果分为：

- 已自动匹配：显示即将填写的内容，可编辑、单项应用或批量应用；
- 待确认：可从已有简历资料中选择内容，再复制或应用；
- 需要手动处理：对文件、密码和复杂控件给出明确提示，不进行危险操作。

默认只填写空字段，不覆盖网页中已有内容。所有结果都先展示给用户，插件永远不会自动提交申请。

### 越用越好用的本地记忆

当你确认某个网页字段对应哪条简历资料，并且填写成功后，插件会记住这次对应关系。下次进入同一招聘网站时，已确认的映射会优先使用，常见字段需要人工处理的次数会逐渐减少。

手动完成自绘下拉框等复杂控件后，也可以点击“检查并记住”。错误记忆可以重新确认、撤销最近一次更新或全部清空。

记忆保存的是网站域名、字段特征摘要和资料字段标识，不保存填写内容、字段原文、完整网页地址、密码或上传文件。

### 规则优先，AI 可选

姓名、邮箱、学校等明确字段优先使用本地中文规则，不消耗 Token。规则无法判断时，可以选择接入自己的 DeepSeek、OpenAI 或其他 `/chat/completions` 兼容接口。

普通表单匹配只向模型发送字段描述和资料字段标识，不发送姓名、电话和经历正文。只有在导入简历时主动启用“AI 精准解析”，提取出的简历全文才会发送到所配置的模型服务。

## 基本使用流程

1. 打开插件，在“简历资料”中录入信息，或导入 PDF、MD、TEX 简历；
2. 保存资料后进入招聘网站，并打开 AutoCV Go 侧栏；
3. 点击“仅扫描”查看全部字段和拟填内容；
4. 确认自动匹配项，并为待确认字段选择合适的简历资料；
5. 应用字段、检查网页，最后由你手动提交申请。

后台自动填写默认关闭。启用后，插件只会在疑似网申页面填写高置信度的空字段，仍然不会自动提交。

## 本地安装

```powershell
git clone https://github.com/Epochex/AutoCV-Go.git
cd AutoCV-Go
pnpm.cmd install
pnpm.cmd build
```

然后：

1. 打开 `chrome://extensions` 或 `edge://extensions`；
2. 开启“开发者模式”；
3. 点击“加载已解压的扩展程序”；
4. 选择项目中的 `.output\chrome-mv3` 目录；
5. 固定 AutoCV Go，点击图标打开侧栏。

更新代码后重新运行 `pnpm.cmd build`，再在扩展管理页点击“重新加载”。

## API 配置示例

以 DeepSeek 为例：

- 接口地址：`https://api.deepseek.com/chat/completions`
- 模型：`deepseek-chat`
- API Key：建议使用单独创建并设置额度限制的 Key

API Key 保存在浏览器本地扩展存储中。项目没有用于接收简历或 API Key 的服务器。

## 隐私与安全原则

- 简历资料和字段记忆默认保存在本地浏览器；
- 默认不覆盖已有内容，不自动上传文件，不填写密码；
- 不自动点击“提交”“投递”等按钮；
- AI 功能由用户主动配置和启用；
- 自动填写模式默认关闭，可随时停用；
- 字段记忆可以撤销或全部清空。

## 当前边界

文件上传、验证码和密码需要手动完成。跨域 iframe、closed Shadow DOM 以及部分高度定制的网页控件可能无法直接操作，但插件仍会尽量将其列为待处理字段。开放 Shadow DOM 已支持扫描。

图片扫描版 PDF 没有可直接提取的文本，建议改用带文本层的 PDF、Markdown 或 LaTeX 文件。页面动态新增教育或项目经历后，需要重新扫描。
